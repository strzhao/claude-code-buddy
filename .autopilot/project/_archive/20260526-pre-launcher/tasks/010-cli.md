# 010: CLI 工具

## 目标

创建 `cli/` 子目录，实现 `buddy-skin upload <directory>` 命令。

## 架构上下文

CLI 是独立 npm 包，本地校验后将目录打包为 zip 上传到 `/api/upload`。校验逻辑从 validation.ts 复制纯校验子集（无 JSZip 依赖的字段检查），服务端始终做完整校验。

## 目录结构

```
cli/
├── package.json
├── tsconfig.json
└── src/
    └── index.ts
```

### package.json

```json
{
  "name": "claude-code-buddy-skin-cli",
  "version": "1.0.0",
  "bin": { "buddy-skin": "./dist/index.js" },
  "scripts": { "build": "tsc" },
  "dependencies": {
    "archiver": "^7.0.0",
    "commander": "^12.0.0"
  }
}
```

## 命令行接口

```
buddy-skin upload <directory> [--server <url>]

Options:
  --server   Store API URL (default: env SKIN_STORE_URL or http://localhost:3000)
```

## 执行流程

1. 读取 `<directory>/manifest.json` → 不存在则报错退出
2. 解析 JSON → 解析失败则报错
3. 本地校验必填字段（步骤 4-9，纯字段检查）
4. 检查 `<directory>/<sprite_directory>/<sprite_prefix>-idle-a-1.png` 存在
5. 使用 archiver 将整个目录打包为 zip (Buffer)
6. POST multipart form 到 `<server>/api/upload`
7. 输出结果:
   - 成功: `✓ Uploaded "Pixel Dog" (id: pixel-dog, version: 1.0.0) — pending review`
   - 校验失败: `✗ Validation failed:\n  - error1\n  - error2`
   - 服务端错误: `✗ Server error: ...`
