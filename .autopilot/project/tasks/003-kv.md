# 003: Vercel KV CRUD + 索引管理

## 目标

创建 `src/lib/kv.ts`，封装所有 KV 读写操作，包括主键 CRUD 和 Redis SET 索引管理。

## 架构上下文

KV 使用复合键 `skin:{id}:{version}` 作为主键。状态索引通过 Redis SET 实现 O(1) 过滤。KV pipeline 用于批量减少 RTT，但非原子事务——失败需幂等重试。

## 需要创建的函数

```typescript
// kv.ts
import { kv } from "@vercel/kv";

export function compositeKey(id: string, version: string): string;
export function kvKey(compositeKey: string): string; // "skin:{compositeKey}"

export async function getSkinRecord(key: string): Promise<SkinRecord | null>;
export async function setSkinRecord(record: SkinRecord): Promise<void>;
// SET + SADD skin-ids + SADD skin-ids:{status}

export async function deleteSkinRecord(key: string): Promise<void>;
// DEL + SREM from all indexes

export async function moveSkinStatus(
  key: string, from: SkinStatus, to: SkinStatus, updates: Partial<SkinRecord>
): Promise<void>;
// SET updated record + SREM old status set + SADD new status set

export async function listSkinsByStatus(status: SkinStatus | "all"): Promise<SkinRecord[]>;
// SMEMBERS + mget, sorted by created_at desc

export async function skinExists(key: string): Promise<boolean>;
// EXISTS

export async function setSkinRecordNX(record: SkinRecord): Promise<boolean>;
// SET NX — 仅不存在时写入，返回是否成功。用于防并发写入
```

## 关键约束

- pipeline 内 SET + SADD 批量执行减少 RTT
- `setSkinRecordNX` 使用 Redis NX 语义防并发
- `listSkinsByStatus` 按 `created_at` 降序排序
- 所有函数使用 `constants.ts` 中的 key 前缀
