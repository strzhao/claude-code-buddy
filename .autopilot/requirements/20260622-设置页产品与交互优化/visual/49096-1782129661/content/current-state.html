当前「设置页」现状（我的理解）

触发链路：菜单栏像素猫咪 → 点击弹 popover（会话面板） → 齿轮按钮 → SettingsWindowController（600×540 浮动 NSPanel，顶部 segmentedControl 切 tab）

设置窗口 · 当前只有 3 个 tab
皮肤 插件 热键

🖼️ NSCollectionView 皮肤网格（已装 + 远程市场下载）

附带：🔊 音效开关 · 🏷️ 始终显示标签

插件 tab：插件列表 + 启用/禁用 + 错误恢复

热键 tab：大号 Recorder + 组合键回显 + 重置默认

✅ UI 里有的

  • 皮肤选择 / 下载 / 切换
  • 插件启用 / 禁用
  • 音效、标签开关
  • 启动器热键录制

❌ UI 里没有（只能 buddy launcher config CLI）

  • AI provider 选择（anthropic / openai 兼容）
  • model 配置
  • API key 输入
  • 开机自启、关于、版本号、反馈

⚠️ 核心矛盾：这个窗口叫「设置」，但实际是「皮肤商店 + 插件管理 + 热键」。真正的 AI 配置反而要敲命令行。普通用户（非开发者）基本配不动 Launcher。
