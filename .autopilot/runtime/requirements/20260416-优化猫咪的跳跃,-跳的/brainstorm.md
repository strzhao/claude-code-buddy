# Brainstorm: 猫咪跳跃优化

## 目标
优化猫咪跳跃系统：跳得更高更远，引入随机变化，增加物理真实性。

## Q&A 结果

### 技术路线选择
- **选择**: 物理引擎驱动（SpriteKit applyImpulse）
- **理由**: 用户希望最真实的物理效果，让重力和初速度自然产生弧线
- **挑战**: 精度控制难，需要确保猫咪不飞出屏幕，落地位置可控

### 视觉增强
- **选择**: 全部都要
  1. 落地 squash/stretch（压扁弹回，重量感）
  2. 跳跃蓄力动画（crouch 下蹲蓄力）
  3. 落地灰尘粒子（小粒子效果）

## 设计方向

### 物理引擎跳跃
- 不再禁用物理体，而是利用 SpriteKit 物理引擎
- 使用 `applyImpulse` 给猫咪施加随机化的冲量
- 重力自然产生抛物线轨迹
- 随机参数：冲量大小（影响高度和距离）、微调水平分量

### 随机系统
- 每次跳跃的初速度在一定范围内随机
- 蓄力时长也随机化（短的快速跳 vs 深蹲大跳）
- 跳跃角度微调

### 视觉反馈层次
1. **蓄力阶段**: 猫咪下蹲（scale Y 压缩），蓄力越久跳越高
2. **起跳瞬间**: scale Y 拉伸（弹射感）
3. **空中**: 正常比例或轻微拉伸
4. **落地 squash**: Y 压缩 + X 拉宽，然后弹回
5. **灰尘粒子**: 落地瞬间在脚底生成小粒子向外扩散

### 关键约束
- 活动边界：跳跃不能让猫咪飞出 activityBounds
- 障碍跳跃：仍需能跳过其他猫咪（检测 + 冲量计算）
- 当前知识：moveBy 动画中断残留位置偏移问题需要注意
- 朝向系统：必须通过 `face(towardX:)` 设置方向
